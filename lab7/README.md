## Описание работы
В данной практической работе №7 изучается параллельная обработка массивов на GPU с использованием технологии CUDA. Реализованы алгоритмы редукции (суммирование, поиск минимума и максимума) и префиксной суммы (scan) с использованием разделяемой памяти GPU. Также выполнено сравнение производительности CPU и GPU и исследовано влияние размера блока потоков на время выполнения.

## Теоретическая часть 
Редукция это операция, которая сводит массив к одному значению (например, сумма, минимум или максимум). Применяется в подсчетах сумм элементов массива, поиске минимального и максимального значения, вычислении среднего значения. На GPU редукция выполняется параллельно, каждый блок обрабатывает свою часть массива а затем частичные результаты объединяются.

Префиксная сумма (scan) это операция при которой каждый элемент выходного массива равен сумме всех предыдущих элементов входного массива. Применяется в фильтрации данных, построении индексов и параллельной сортировке.

## Практическая часть
#### Задание 1: Реализация редукции
В данном задании была реализована редукция суммы массива на GPU с использованием CUDA. Каждый блок потоков считает частичную сумму, после чего эти суммы снова редуцируются. Результат сравнивается с вычислением на CPU. Результаты показали, что сумма на GPU совпадает с суммой на CPU с небольшой погрешностью.

#### Задание 2: Реализация префиксной суммы 
Во втором задании реализован алгоритм Blelloch exclusive scan. Сначала сканирование выполняется внутри каждого блока, затем считается скан для сумм блоков и после этого добавляются смещения. Результаты GPU совпадают с CPU-реализацией с допустимой погрешностью.

#### Задание 3: Анализ производительности 
В третьем задании было измерено время выполнения редукции и scan на CPU и GPU для массивов разного размера. Также использовались разные размеры блока потоков. По результатам видно, что для маленьких массивов CPU работает быстрее, для больших массивов GPU значительно быстрее CPU, размер блока потоков влияет на время выполнения.

#### Дополнительное задание
В дополнительных заданиях были реализованы редукция для поиска минимума и максимума, проверка корректности алгоритма scan на большом массиве, измерение времени работы GPU при разных размерах блоков.

## Ответы на контрольные вопросы
1. В чём разница между редукцией и сканированием?
Редукция это операция при которой весь массив сводится к одному значению, например сумме, минимуму или максимуму. Сканирование это операция при которой для каждого элемента массива вычисляется сумма всех предыдущих элементов.

2. Какие типы памяти CUDA используются для оптимизации редукции и сканирования?
Глобальная память для хранения входных и выходных данных, разделяемая память для ускорения вычислений внутри блока так как она работает быстрее и позволяет уменьшить количество обращений к глобальной памяти.

3. Как можно оптимизировать префиксную сумму на GPU?
Префиксную сумму можно оптимизировать используя разделяемую память, уменьшая синхронизации, применяя эффективные алгоритмы такие как Blelloch scan.

4. Приведите пример задачи, где применяется сканирование.
Сканирование используется при фильтрации массивов когда нужно определить новые позиции элементов, например при удалении элементов по условию.
